Description


This tool is a rig helper. It's used in Maya to help with control objects and icon creation, and other basic rig construction tasks. The Curves Tool script contains commonly used features such as :
- Control objects creation, with various shapes to choose from. Creation of multiple objects at once.
- Colorizing options.
- Preset and custom attribute creation.
- Control clean up options such as zeroing out the curves using freeze transformations. Lock and hide made easy and quick!
This script was first developped by Jennifer Conley in 2012 and then by myself : David Saber, since 2022.


V4.03 : first public release since the Python re-write started:

With Maya 2025, the V3.x Pymel RigBox script does not run anymore. For my students, I had to speed up the developpment of V4.x, a full Python, non-Pymel version. I got the main tools up and running :
- curve creations
- curve coloring
- lock and hide
- object naming.
This is the first public release, the essential functions are working, but some others are not. I hope to restore all tools in the future.



Installation / Uninstallation


V4 by David, Pymel not required:

1) Unpack the folder "RigBox" and copy it inside the user folder: Documents\maya\scripts
2) Open RigBox_Launcher.py with a text editor, and copy all text inside
3) In Maya, in the rigging shelf, open the shelf editor: the gear in the bottom left.
4) In the "shelf content" column, use the "new item" button to add a new button, then choose its position in the shelf using the "move up" and "move down" buttons.
5) With the "user script" selected, go to the command tab, first choose "Python" radio button. Then paste the code you copied from RigBox_Launcher.py.
6) Back to the "shelves" tab, you can personalise the button by choosing the RigBox.png as the icon name, setting up the tooltip and the icon label, etc.
7) Click the button "Save all shelves", this will close the shelf editor
8) Click the button to start the script.
9) To uninstall, delete the button and the script folder.

V3 by David, Pymel required:

For those still using the V3.x Pymel script:

A) HOW TO INSTALL: PYMEL
1) First install Pymel in Maya 2022 (or check if it's already installed: read next line). PyMEL is an optional item when you install Maya 2022, you could run the installer again and tick the PyMEL box to install it. If you didn't install it during the Maya installation process, follow the instructions below.
2) Lets check if its really installed: navigate to maya installation folder, usually located at C:\Program Files\Autodesk\Maya2024. Find the folder "bin", select it, and while holding down shift press RMB (right mouse button) to open modified context menu, from there find and click "open in powerShell" or "open in terminal".
3) In Powershell or Terminal, you should see this:
PS Maya2022\bin>
4) You must use this command :
.\mayapy.exe -m pip list
To get this command, just type mayapy and press [TAB] (it will expand like .\mayapy.exe) then continue typing " -m pip list". then you should see this:
PS Maya2022\bin> .\mayapy.exe -m pip list
5) If you see this, press enter
6) If you get something like this, it means there is no pymel:
future 0.18.2
pip 20.2.2
PySide2 5.15.2
setuptools 41.2.0
shiboken2 5.15.2
shiboken2-generator 5.15.2
7) Then you need to install Pymel this way:
.\mayapy.exe -m pip install pymel
8) After installation is complete you can check it again, using :
.\mayapy.exe -m pip list
9) If you get something like this, it means Pymel is now installed:
future 0.18.2
pip 21.2.2
pymel 1.4.0
PySide2 5.15.2
setuptools 41.2.0
shiboken2 5.15.2
shiboken2-generator 5.15.2
10) You're ready to go! no need to modify PATH environment variable.
11) If you receive message that new version of pip installer is available, and if you wish to upgrade it, use this cmd:
.\mayapy.exe -m pip install --upgrade pip
12) That's it. For more info, You can also check this discussion : https://forums.autodesk.com/t5/maya-programming/maya-2022-pymel-how-to-import/m-p/10409310

B) HOW TO INSTALL: RIGBOX SCRIPT
Now install Rigbox in Maya 2022-2023 following these instructions below.
1) Go to your Maya preference folder, usually it's in Documents\Maya. Go inside the folder Documents\Maya\scripts .
2) Unzip/Unrar the file "Rigbox Reborn Curve Tool Python 3.rar", place the "RigBox3" folder into the folder Documents\Maya\scripts
3) Once the files have been placed correctly, start or restart Maya.
4) Copy and paste the following lines of code into the Script Editor, Python tab, and click the Execute button.
import RigBox3.rr_main_curves
RigBox3.rr_main_curves.window_creation()
5) A Rigbox Windows should popup.
6) Get the script as a permanent button in Maya's interface: in Maya, display the Rigging tool tab. In the script editor, select these 2 Python lines. Middle click and drag them to the right end of the Rigging tool tab. An ugly button should appear.
7) Right click the button and choose edit. Go to the "Shelves" tab. Click the folder button on the right side of "Icon name", browse to where you put the rigbox files. Pick the file: "rr_RigBox3_icon.png". You can also type "RigBox" in ""Icon label".
8) Voila! Cheers, Jen and David



General Usage Instructions


Starting

To start the process, click the RigBox button.
Follow the info video, it's an old video made by Jen, but it's still useful. DreamCraft Link. Vimeo Link.

Creation

Unfold the "Creation" section:
This is a custom GUI to easily create commonly used control icons for rig construction. Includes 2D, 3D, and text based curves. It provides features for:
- Priming controls based on a selected transform node, or a hierarchy of nodes.
- Naming control objects.
- Naming priming groups
- Constraining selected transform node, with Parent or Orient constraint options
How to:
- Optional: enter a name, a prefix and a suffix, for the object that will be created. If you don't enter text, the object you will create at a later step will get the clicked button's name.
- Optional: manage snapping and grouping options: read below.
- Click a button to choose a curve object with a personalized shape.
- The object will be created at the center of the scene, or snapped to other object if you set the snapping and grouping options.

Grouping

Unfold the "Creation" section. Under "Control naming", in the right drop down, choose "group".
Sometimes you need to zero-out control objects of which the axis must remain tilted. A good example is fingers setup. A traditional method in Maya is to use double grouping. The method to do this is described in Jennifer Conley's video about RigBox Curve Tool (check link above, under "Starting").

Here's how you would do it manually for a control object that may constrain a target object :
- Create a target object that is tilted, such as an arm joint.
- Using RigBox, create a control object and name it as such.
- Place the control object on the target object such as the bone root.
- Freeze transforms the control object.
- Control object becomes child to the target object.
- Double group the Control object: CTRL+G, CTRL+G.
- Unparent the top group to put it at the Outliner root with SHIFT+P.
- Control object : freeze transforms it again.
- The Control object now has the same local axis as the object to constrain.

RigBox will automate things for you :
- Create an target object that is tilted, such as an arm joint.
- Using RigBox, create a Control object.
- Place the Control object on the target object such as bone root.
- Under grouping options, left drop down: choose if you want to adapt a single object, or create then adapt multiple objects on a full joint chain, except on the last joint.
- Under grouping options, middle drop down: choose if the multiple objects to be created won't have a parent ("default" setting: they'll be all at the Outliner's root) or if they will be organised in a parent child hierarchy. The "Hierarchy" option needs the "chain" option to be selectd in the left drop down.
- Under grouping options, right drop down: choose if you want this Control object to constrain the target object, in Orient or Parent.
- Select Control Object, shift select target object. The curve must be selected first, then the transform node. This selection order was choosen because it replicates actual parenting inside of Maya.
- Click apply : The Control object now has the same local axis as the target object.
- Under "Control naming", in the right drop down, if you keep "group" selected, it will always obey the rules you set in the "Grouping Options" section.

Snapping

Unfold the "Creation" section. Under "Control naming", in the left drop down, choose "snap".
Create an object to snap the RigBox control to. Move and rotate it. Select this object. Click a curve creation button. The Control curve's translation, rotation (but not the scale), will match the selected object.


Coloring

Unfold the "Coloring" section:
It's a script to quickly color control icons for a rig. Able to be run on a selection. Also has options for templating an object.
Once the object is created, click a button to color it.
Click the buttons for templating the object. The "template attr" button will add a channel in the channel box that will allow you to switch between templated and untemplated.

Attributes

Unfold the "Attributes" section:
This section will create attributes, also called "customized channels".
It's a custom GUI to easily add common attributes to objects. The "Preset Options" part will add a bunch of channels. For example, the "IK Foot" button will add custom channels such as "Foot Roll" and such. But the connections such as driven keys must be setup manually.
The "Custom Options" part allows for additonal, custom attributes, to be added to objects without having to navigate to other windows. Its "Create" button will add a single attribute in the channel box. First choose a data type : float, integer, boolean or "Sep". Then Type a name for the new channel. Then choose a min and max value. Then click Create.
The "Sep" data type isn't really a data type. It will create a dotted line separator in the channel box.

Lock and hide

Unfold the "Lock and hide" section:
A custom GUI to easily lock and hide unused transform channels of a control icon. This will lock and hide channels according to the check boxes. It will also make these channels visible in the channel box after they've been hidden.
Able to use preset options for FK and IK controls: for example, if you check the "IK" radio button, it will uncheck translation channels and will check all other channels, which is typical for IK control objects.
This script function is MUCH more handy than the default Maya process.

Development history


Development by Jennifer Conley, Version 0.x to 1.1.2 : DOWNLOAD

Jen is the one who created the script in 2012 and developped it for a long time, using the Pymel language. To check all her updates, read the change log in the version history section below.
The "Curve Tool" was a small part of a much larger project. The "RigBox" scripts were a collection that included several other scripts, such as "RigBox Reborn - Layout Tool for Maya" or "RigBox_Reborn - Hip Averaging for Maya", etc. They were all available at Highend 3D, in the Jennifer free tools page.

Development by David Saber, Version 3.01 and later : DOWNLOAD

In older Maya version, I was using only the "curve tool" from the Rigbox collection. But since Maya 2022, Autodesk isn't developping Pymel anymore, and Pymel isn't installed by default with a new Maya install. However, Pymel relies on Python to run scripts, and, begining with Maya 2022, Python 3 replaced Python 2. This broke compatibility with old script such as RigBox.
So, in 2021, a Reddit user called SheepRSA updated the Curve Tool script to make it compatible with Maya's latest versions. For that, he converted the Python language from version 2 to version 3. Some parts of the RigBox buttons were'nt working anymore as the script wasn't tested much. But the main tools were working, and were compatible with Maya 2022, 2023 and 2024, IF Pymel was installed on top of Maya.
At that time, I informed Jen that I could send her this updated script, so she could upload it to her Highend account. But she never answered. Unfortunately, I believe Jen quit the 3d business since a long time. So I uploaded the script to my own Highend account because I know many Maya riggers were waiting for RigBox in Python 3! After that, I noticed many people downloaded the updated script and asked for bug fixes. So I kept updating the script from time to time and I also fixed some bugs that existed since the Jennifer era.
Thanks to Jennifer Conley for creating the RigBox tool 12 years ago! It is an awesome tool for rigging.

Development by David Saber, Version 4.00 and later : DOWNLOAD

When Maya 2025 appeared, installing Pymel was not enough to run old Pymel-based scripts. There is a way to get it to work but it seems complicated. No reason to spend time trying to keep alive an old language. So it was time for Rigbox Version 4, which aims to get rid of the mandatory Pymel installation. Having a pure Python-based script is also better to pursue its development. I started working on a full Python translation in january of 2024. In September, as my students needed this tool on Maya 2025, I sped up the developpment to get the main tools up and running : curve creations, curve coloring, lock and hide, object naming. So the first public release is v4.03. The other RigBox function may be buggy or not working but I hope to fix them in the future.
